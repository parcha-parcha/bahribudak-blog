# DOSYA VE BELGE ADLANDIRMA STANDARDI

**Belge kodu:** BB-SYS-STD-001  
**Revizyon:** R00  
**Yürürlük tarihi:** 25 Haziran 2026

## 1. Teknik yayın belge kodları

```text
BB-ORG-TP-NNN   Örgü / Knitting
BB-BOY-TP-NNN   Boya / Dyeing
BB-APR-TP-NNN   Apre / Finishing
BB-ORT-TP-NNN   Ortak Teknik Yönetim
```

İngilizce yayınlarda proses kısaltmaları uluslararası karşılıkla kullanılabilir:

```text
BB-KNT-TP-NNN
BB-DYE-TP-NNN
BB-FIN-TP-NNN
```

Her dildeki karşılık aynı yayın numarasını temsil etmelidir.

## 2. Revizyon kodu

- İlk yayın: `R00`
- Teknik düzeltme: `R01`, `R02`, ...
- Yalnızca yazım düzeltmesi yapılmışsa revizyon notunda belirtilir.
- Formül, standart, proses önerisi veya ölçüm yöntemi değişmişse mutlaka revizyon artırılır.

## 3. Web dosya adı

Web sayfası ve görsellerde küçük harf, ASCII karakter ve tire kullanılır:

```text
ramoz-kompaktor-boyutsal-stabilite.mdx
reaktif-boyama-ph-yonetimi.jpg
```

Boşluk, Türkçe karakter, parantez ve sürüm ifadesi web slug içinde kullanılmaz.

## 4. İndirilebilir teknik dosya

```text
BB_<ALAN>_<TUR>_<NUMARA>_<KONU>_R<REVIZYON>_<YYYY-MM-DD>.<uzanti>
```

Örnek:

```text
BB_APR_FRM_012_Ramoz_Proses_Kontrol_R01_2026-06-25.xlsx
BB_BOY_TN_004_Reaktif_Boyama_pH_Yonetimi_R00_2026-06-25.pdf
```

## 5. Dosya türü kısaltmaları

- `TP`: Teknik Yayın / Technical Publication
- `TN`: Teknik Not / Technical Note
- `FRM`: Form
- `CHK`: Kontrol Listesi / Checklist
- `CAL`: Hesaplama Aracı / Calculation
- `TRN`: Eğitim Notu / Training Note
- `PRO`: Prosedür / Procedure
- `STD`: Standart / Standard

## 6. Görsel standardı

- Teknik yayın kapağı: 1600 × 900 px
- Sosyal medya seri görseli: kullanılan platformun güvenli alanına göre hazırlanır
- Resmî logo dışında amblem yeniden çizilmez
- Poppins dışında ikinci bir yazı ailesi kullanılmaz
- Ana renkler kurumsal kimlik v1.1 paletinden seçilir
